## Multi-tenancy model

Every table has a `garageId` column.

- **Super Admin** вЂ” no `garageId` on their `Profile`; can query across all garages.
- **Ops Manager** вЂ” `Profile.garageId` is set at account creation; can only see/modify their garage.
- **Mechanic** вЂ” same garage scope as their ops manager; sees only their own jobs.

Enforcement happens at two layers:
1. **Application layer** вЂ” every service method accepts `garageId` and filters on it.
2. **Database layer** вЂ” Supabase Row Level Security (RLS) policies block any query that doesn't match the caller's `garageId` claim in the JWT.

## Inventory audit rule

Every stock or price change **must** produce an `InventoryAuditEntry` row.  
Do **not** call `prisma.inventoryItem.update()` directly вЂ” always go through  
`InventoryService.addStock()` or `InventoryService.updatePrice()`.  
Auto-deductions from invoice raises use `InventoryService.deductStock()` which  
sets `comment` automatically (no human input required).

## Adding a new payment provider

1. Create `src/lib/plugins/payments/razorpay.ts` implementing `PaymentProvider`.
2. Add `case "razorpay": _provider = new RazorpayPlugin();` in `index.ts`.
3. Set `PAYMENT_PROVIDER=razorpay` in `.env.local` / Vercel dashboard.
4. Done вЂ” no service or route handler changes needed.

## Getting started (first time)

```bash
# 1. Fill in .env.local (DATABASE_URL, DIRECT_URL, SUPABASE_SERVICE_ROLE_KEY, etc.)

# 2. Generate the Prisma client from the schema
npx prisma generate

# 3. Push the schema to your Supabase database (creates all tables)
npx prisma db push

# 4. (Optional) Seed with sample data
npx prisma db seed

# 5. Start the dev server
npm run dev
```
